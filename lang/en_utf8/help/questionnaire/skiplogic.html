Skip Logic & Conditional branching

Respondents can be routed through a questionnaire using 2 types of branching logic:

1. Skipping specific questions

A skip pattern will jump a respondent over a group of questions that isn't relevant to them. Use a Yes/No question for this.

Example 1

  1. Page 1
    • Question 1 Do you own a car? Yes/No.
      • Answer Yes -> go to page 2
      • Answer No -> go to page 3 (i.e. skip over irrelevant question(s) in page 2
  2. Page 2
    • Question 2: What is the color of your car?
    • Question 3: What is the make of your car?
    • Question 4: How long have you owned your car?
  3. Page 3
    • Question 5: Where do you work?
    • etc.

Procedure to be followed to obtain the above "skip logic" flow

  • Create Q1 as a Yes/No question, and set Skip Logic to "Yes" and set Response Required to Yes, of course!
  • Insert a Page Break ---
  • Create Q2 as a radio button or text or dropdown question type
  • Create Q3
  • Create Q4 (you can create as many questions as you like on that page)
  • Insert a Page Break---
  • Now continue creating questions normally, until you want to use Skip Logic again.

2. Conditional branching:

A branch pattern will route a respondent to the appropriate section of the questionnaire: each respondent follows one of the branches. Use a Radio Button or Dropdown Box question for this.

Example 2

  1. Page 1
    • Question 1: Where do you teach? Primary school / Secondary education sector / University.
      • Answer: Primary school -> go to page 2
      • Answer: Secondary education sector -> go to page 3
      • Answer: University -> go to page 4
  2. Page 2
    • Question 2: How long have you been a Primary school teacher?
    • Question 3: What age groups to you teach? [go to page 5]
  3. Page 3
    • Question 4: How long have you been a Secondary school teacher?
    • Question 5: What it your main subject [go to page 5]
  4. Page 4
    • Question 6: How long have you been a University professor?
    • Question 7: What is your domain of research? [go to page 5]
  5. Page 5
    • Question 8: Tell us what you think of Moodle in your everyday teaching routine...
    • etc.

Procedure to be followed to obtain the above "conditional branching" flow

  • Create Q1 as a Radio Button or Dropdown Box question, and set skip logic to "Yes" and set Response Required to Yes, of course!
  • Insert a Page Break ---
  • Create Q2 as a Text Box or Numeric question
  • Create Q3 as a Text Box question
  • Insert a Page Break ---
  • Create Q4 as a Text Box or Numeric question
  • Create Q5 as a Text Box or Radio Button or Dropdown Box question
  • Insert a Page Break---
  • Create Q6 as a Text Box or Numeric question
  • Create Q7 as a Text Box question
  • Insert a Page Break---
  • Now continue creating questions normally, until you want to use Skip Logic again.

Note 1

It is not possible to embed skip logic pages within pages which are already part of a skip logic flow. That is to say that the following example flow is not possible in the current version of the Moodle Questionnaire:

*Example 3 (not possible)

  • Page 1
    • Question 1 Do you own a car? Yes/No.
      • Answer Yes -> go to page 2
      • Answer No -> go to page 5
  • Page 2
    • Question 2: Did you buy your car new? Yes/No.
    • Answer Yes -> go to page 3
    • Answer No -> go to page 4
  • Page 3
    • Question 5: What is the make of your car?
  • Page 4
    • Question 6: What is the color of your car?
  • Page 5
    • Where do you work?

Note 2

A respondent can walk a questionnaire which contains one (or more) skip logic pages forward and backward, as any "normal" multi-pages questionnaire. This means that any answer can be changed at any time (until the Submit button is pressed on the final page). This means that changing skip logic answers will have some implications, as shown in the example below. The same would apply to "conditional branching" questions, of course.

Example walk-through with Example 1 above.

  1. Page 1 Question 1 Do you own a car? Yes/No. Answer: Yes
  2. Page 2 Question 2: What is the color of your car? Answer: Blue
    • Question 3: What is the make of your car? Answer: Rolls-Royce
    • Question 4: How long have you owned your car? Answer: 20 years
  3. Page 3 Where do you work? Answer: I work from home.
  4. Previous page -> Page 2
  5. Previous page -> Page 1; Do you own a car? Yes/No. Answer: Yes change Answer to No.
  6. Next page -> Page 3 Question 5: Where do you work? Answer: I work from home.

In step 5, respondent changed his previous Yes answer to No, so page 2 with its questions 2 to 4 is skipped over and any previously-entered responses to those questions are deleted (from the Moodle site's database).
