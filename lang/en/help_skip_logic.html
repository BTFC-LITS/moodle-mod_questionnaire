Skip Logic & Conditional branching

The Skip logic (aka Conditional branching) feature will jump a respondent over a group/page of questions that isn't relevant to them.

Skip logic simple example

  1. Page 1
    • Question 1 Do you own a car? Yes/No.
      • Answer Yes -> go to page 2
      • Answer No -> go to page 5 (i.e. skip over irrelevant questions in pages 2 through 4)
  2. Page 2
    • Question 2: What is the color of your car?
    • Question 3: Did you buy it new or second-hand?
      • Answer new -> go to page 3
      • Answer second-hand -> go to page 4
  3. Page 3
    • Question 4: Why did you buy your car new?
  4. Page 4
    • Question 5: Why did you buy your car second-hand?
  5. Page 5
    • Question 6: Do you sometimes use public transport to go to work?
    • etc.

Procedure to be followed to obtain the above "skip logic" flow

  • Create Q1 as a Yes/No question.
    • Question Name: own car. Note that you must give your question a name, as this name will be used later on in the Questionnaire for the list of Depend questions. Please make this name as short as possible.
    • Response Required: Yes
    • Question Text: Do you own a car?
  • Insert a Page Break ---
  • Create Q2 as a Text Box question type
    • Question Name: optional
    • Response Required: Yes or No
    • Depend Question : own car->Yes
    • Question Text: What color is your car?
  • Create Q3 as a Radio buttons or Dropdown box question type
    • Question Name: car status
    • Response Required: Yes
    • Depend Question : own car->Yes
    • Question Text: Did you buy it new or second-hand?
    • Possible answers:
      • new
      • second-hand
  • Create Q4 as a Text Box question type
      • Question Name: optional
      • Response Required: Yes or No
      • Depend Question : car status->new
      • Question Text: Why did you buy your car new?
  • Insert a Page Break---
  • Create Q5 as a Text Box question type
      • Question Name: optional
      • Response Required: Yes or No
      • Depend Question : car status->second-hand
      • Question Text: Why did you buy your car second-hand?
  • Insert a Page Break---
  • Question 6 (see above). Now continue creating questions normally, until you want to use Skip Logic again.

Note 1

A respondent can walk a questionnaire which contains one (or more) skip logic pages forward and backward, as any "normal" multi-pages questionnaire. This means that any answer can be changed at any time (until the Submit button is pressed on the final page). Obviously if a respondent changes their response to a question containing a "Depend Question" setting, this will take the respondent along a new, different path through the questionnaire.

Note 2

Once a questionnaire has been created containing "Depend Questions", moving or deleting those questions must only be done with great care, not to destroy the Skip Logic flow. Should you move a question that has a "Depend Question" setting before that Depend Question, it's "Depend Question" setting will be removed. If you delete a question that is linked to as a Depend Question by other questions, the "Depend Question" settings in those questions will be removed.

Note 3

Only the following 3 "choice" question types can be used to trigger the Skip Logic feature: Yes/No, Dropdown boxes and Radio buttons. However, as seen in the example above, any question type can use a Depend Question setting once they have been made available by the 3 "choice" question types.

Note 4

On the Questions editing page, for those questions which use a Depend Question setting, the Depend Question name and choice will be displayed in red.
